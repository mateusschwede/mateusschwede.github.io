Git e Github base

Sintaxe básica Git e Github

Resumo


Sistema de controle de versão distribuido: clone do repositório na máquina do desenvolvedor

Projetos divididos em 3 estágios
1 - Working Directory (diretorio de trabalho, pêgo do repositório para a máquina do desenvolvedor)
2 - Staging Area (sala de espera, ficam antes de serem envidados ao repositório)
3 - Git Directory (repositório central)
WORKING DIRECTORY -> git add -> STAGING AREA -> git commit -> GIT DIRECTORY

Configuração Inicial Git Bash:
> git config --global user.name "mateus"
> git config --global user.email "mateus1908.schwede@gmail.com"
> git status (status do repositório)

Ignorar Arquivo: criar arquivo vazio .gitignore, com o nome dos arquivos que serão ignorados
ex: .gitignore, mateus.txt, umapasta/


========== LOCAL ==========

Configurar local de trabalho do git: ir no icone do bash, com o botão direito e alterar o endereço de inicialização
Criar Repositório: Entrar na pasta com 'cd', > git init
Inserir Arquivos no Repositório> git add .
Remover Arquivos do Repositório(que foram deletados)> git rm arquivo.php
Concluir Alterações(Commit)> git commit -m "alteracao01"

Consulta Staging Area> git diff --staged
Consulta Working Directory> git diff
Consulta Commits> git log
Consulta Commits detalhado> git log -p (Q para sair)
Consulta Commits Limite> git log -p -1 (1 só commit no log)
Consulta com x Commits> git log --pretty=oneline

Editar Commit Atual> git commit --amend -m "ultimocommit"
Remover da Staging Area> git reset HEAD arquivo.php
Reverter Alterações de arq. ñ commit, nem add> git checkout -- arquivo.php

TAG: atalho para determinado status do sistema. ex: 1.0, 1.1, 1.2...
Criar tag> git tag -a nometag -m "inicial"
Criar tag commit antigo> git tag -a nometag idcommit -m "inicio"
Listar tags> git tag | > git show nometag (com detalhes)
Consulta por tags> git checkout nometag
Voltar a tag padrão> git checkout master
Deletar tag> git tag -d nometag

BRANCH (Ramificações): master é branch padrão
uso: implementar nova funcionalidade em paralelo com a master
Criar e Selecionar branch> git checkout -b nomebranch
Voltar ao master> git checkout master
Listar branchs> git branch
Deletar branch> git branch -d nomebranch

MERGE: passar coisas do branch x pro master
> git merge branchantigo (branch que estou será o destino)


========== GITHUB ==========

CLONE: Usar Repositório Existente
PUSH: Enviar alterações
FETCH: Busca diferenças do ramo atual, sem alterar o branch
PULL: Baixar código atualizado / atualizar repositório (fetch+merge das diferenças / Usado após um Fetch indicar que houve alterações)
PULL REQUEST: Pedidos de melhorias(pulls) de vários programadores. Requisitar atualização do repositório, colocando um fork ao master projeto, por exemplo
FORK: Clonar o repositório que deseja editar pra sua conta no github, para usá-lo por conta própria
ISSUE: Comentário informando erro, nova feature ao repositório... Evite issues duplicadas caso sua sugestão/dúvida já esteja presente!

1- Conectar pc com um par de chaves ssh ao github, gerado pelo bash
> ssh-keygen
2- Pegar key gerada na pasta .ssh: abrir arquivo.pub com notepad, copiar, e lá no github, em settings, ir no ssh, adicionar nova e colar a key do .pub(com isso, poderei gerenciar o github com o bash)

3- Criou-se o repositório no github, aí segue... copiar o ssh pro bash
> git clone seussh nomepastaarmazenamento
ex: (arquivos criados na sua pasta local...), editar arquivos ir, com o bash, para a pasta local
> git add .
> git commit -m "primeirocommit"
> git push origin(seuremote) master

4- Fazer outro repositório, para edições, com base no primeiro
> git clone seussh nomepastaarmazenamento
após, ir na pasta com o bash

5- Editar arquivos e após:
> git add .
> git commit -m "segundo commit"
> git push origin(seuremote) master

6- Voltar ao repositório1, pra atualizar os arquivos
> git pull origin master

BAIXAR DO GITHUB: (Clone)
1- New -> new Repository -> inserir nome, descrição e marcar caixa "readme.md"
2- Clone or Download -> copiar Ssh Key / Https
3- No caso de Ssh, copie a key do arquivo .pub gerado!
4- Bash >git clone keycopiada/httpscopiado

IMPORTAR AO GITHUB:
1- Mover seus arquivos à pasta baixada
2- > git add .
3- > git commit -m "import 06/07/2019"
4- > git push //enviar ao github

GITHUB EM EQUIPE (Enviar):
1- Criar novo bransh> git checkout -b nomebransh
2- Criar arquivos ao novo bransh> git add .
3- Commit> git commit -m "novos arquivos"
4- Importar ao Github> git push
5- Github abrirá e solicitará o Pull Request, informe assunto e descrição e confirma
Baixar código atualizado:
1- Fazer um pull do código> git pull


========== COLABORAÇÃO GITHUB ==========

a) Maioria dos projetos possuem arquivo "contributing.md", que especifica as regras de contribuição e principais necessidades do projeto
b) É importante, além da descida e subida do novo código, manter um documento detalhado de todo seu trabalho nessas aventuras
c) A fachada de entrada dos projetos é o arquivo "readme.md", portanto capriche na sua edição:
1º Descrição detalhada do projeto / funcionalidades / telas do projeto
2º Diário de bordo (data e descrição em tópicos ao longo do seu trabalho/dificuldades/futuros trabalhos)

1- Copiar url do projeto que quero editar, fazer fork dele, no botão fork
2- Clonar o projeto pro pc local, copiando o ssh
> git clone meusssh nomepastaarmazenamento

3- Editar arquivos e após:
> git add .
> git commit -m "funcionalidades adicionadas"
> git push origin master

4- Enviar ao autor: tela do projeto, clicar no Pull Request, new Pull Request, create Pull Request
5- Autor receberá um pull request do projeto pra analisar, se aceitar é só clicar no Merge Pull Request
